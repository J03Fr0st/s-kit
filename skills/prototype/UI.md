# UI Prototype

Use a UI prototype for layout, visual hierarchy, flow, and interaction questions.

## Shape

- Use the project's existing app/router conventions.
- Put variations behind a query parameter, local toggle, or clearly marked prototype route.
- Make variations materially different, not cosmetic color swaps.
- Keep fake data local and obvious.
- Avoid production wiring, persistence, auth changes, or API changes unless those are the question.

## Verification

Before presenting the prototype:

- Start the existing dev server if needed.
- Open the route in a browser.
- Check desktop and mobile widths.
- Confirm controls and variant switching work.

## End State

Capture the chosen direction in the design or issue. Delete the route or fold the selected pattern into production code when implementation starts.
