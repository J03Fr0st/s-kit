# Logic Prototype

Use a logic prototype for state machines, business rules, parsing, scheduling, or workflow questions.

## Shape

- Prefer a tiny CLI or terminal app.
- Print the full relevant state after every action.
- Include a small menu or scripted examples for edge cases.
- Keep data in memory unless persistence is the question.

## Minimum Output

At the top of the prototype file or README, state:

- The question being answered.
- The command to run.
- What result would prove the design works or does not work.

## End State

When done, record the verdict somewhere durable and delete the prototype unless the user explicitly asks to keep it for another round.
