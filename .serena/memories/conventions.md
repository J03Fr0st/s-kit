# Conventions

- Treat the repo as a workflow kit/documentation seed until a concrete tool or app structure is added.
- Prefer Markdown for new durable workflow guidance unless the user asks for runnable tooling.
- Do not invent package managers, test runners, or language-specific structure before project files establish them.
- Keep `.serena/project.local.yml` local-only; shareable Serena settings belong in `.serena/project.yml`.
